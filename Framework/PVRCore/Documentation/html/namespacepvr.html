pvr Namespace Reference

Main PowerVR Framework Namespace. More...

Namespaces

 Api
 Enumeration of all API types supported by this implementation.
 
 api
 Main PVRApi Namespace. In the PVRCore projects, contains interfaces and enumerations that conceptually belong to PVRApi but need to be shared with other PVR Framework projects (GraphicsContext interface, enums)
 
 ApiObjectType
 Enumeration containing all possible API object types (Images, Buffers etc.).
 
 assets
 Main PVRAssets namespace. In the context of PVRCore, contains enumerations that conceptually belong to PVRAssets but are shared between different PowerVR Framework projects.
 
 BorderColor
 Enumeration describing default border colors for textures.
 
 Capability
 Pre-defined Capability presense values.
 
 ColorSpace
 Enumeration of Colorspaces (Linear, SRGB).
 
 ComparisonMode
 An enumeration that defines Comparison operations (equal, less or equal etc.). Especially used in API classes for functions like depth testing.
 
 CubeFace
 Enumeration of the six faces of a Cube.
 
 DataType
 An enumeration that defines data types used throughout the Framework. Commonly used in places where raw data are used to define the types actually contained.
 
 DeviceQueueType
 Enumeration of all types of DeviceQueue.
 
 IndexType
 An enumeration that defines a type that can use as an index, typically 16 or 32 bit int. Especially used in Model classes.
 
 LoadOp
 Enumeration all possible values of operations to be performed on initially Loading a Framebuffer Object.
 
 math
 Contains mathematical functionality and classes, such as bounding box calculations, intersections etc.
 
 native
 Contains functionality used to interface with the underlying native platform.
 
 PrimitiveTopology
 Enumeration containing all possible Primitive topologies (Point, line trianglelist etc.).
 
 Result
 Pre-defined Result codes (success and generic errors).
 
 SamplerFilter
 Enumeration describing a filtering type of a specific dimension. In order to describe the filtering mode properly, you would have to define a Minification filter, a Magnification filter and a Mipmapping minification filter. Possible values: Nearest, Linear, None.
 
 SamplerWrap
 Enumeration for defining texture wrapping mode: Repeat, Mirror, Clamp, Border.
 
 ShaderType
 Enumeration of all supported shader types.
 
 StoreOp
 Enumerates all possible values of operations to be performed when Storing to a Framebuffer Object.
 
 strings
 Contains several valuable helpers to assist with common string operations: Starts with, ends with, create with printf-style formatting and others.
 
 system
 Contains functionality for the interaction of the Framework with the system it runs on (Console etc).
 
 TextureDimension
 Enumeration of Texture dimensionalities (1D/2D/3D).
 
 utils
 Contains assorted utility functions (test endianness, unicode conversions etc.)
