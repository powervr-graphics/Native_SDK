388 /*!***************************************************************************
389  @class PVRShell
390  @brief Inherited by the application; responsible for abstracting the OS and API.
391  @details PVRShell is the main Shell class that an application uses. An
392  application should supply a class which inherits PVRShell and supplies
393  implementations of the virtual functions of PVRShell (InitApplication(),
394  QuitApplication(), InitView(), ReleaseView(), RenderScene()). Default stub
395  functions are supplied; this means that an application is not
396  required to supply a particular function if it does not need to do anything
397  in it.
398  The other, non-virtual, functions of PVRShell are utility functions that the
399  application may call.
400  *****************************************************************************/

425  /*!***********************************************************************
426  @brief Initialise the application.
427  @details This function can be overloaded by the application. It
428  will be called by PVRShell once, only at the beginning of
429  the PVRShell WinMain()/main() function. This function
430  enables the user to perform any initialisation before the
431  render API is initialised. From this function the user can
432  call PVRShellSet() to change default values, e.g.
433  requesting a particular resolution or device setting.
434  @return true for success, false to exit the application

443  memory/resources acquired in InitApplication().
444  @return true for success, false to exit the application

446  virtual bool QuitApplication() { return true; };
447 
448  /*!***********************************************************************
449  @brief Initialise the view.
450  @details This function can be overloaded by the application. It
451  will be called by PVRShell after the OS and rendering API
452  are initialised, before entering the RenderScene() loop.
453  It is called any time the rendering API is initialised,
454  i.e. once at the beginning, and possibly again if the
455  resolution changes, or a power management even occurs, or
456  if the app requests a re-initialisation.
457  The application should check here the configuration of
458  the rendering API; it is possible that requests made in
459  InitApplication() were not successful.
460  Since everything is initialised when this function is
461  called, you can load textures and perform rendering API functions.
462  @return true for success, false to exit the application
463  *************************************************************************/
464  virtual bool InitView() { return true; };
465 
466  /*!***********************************************************************

468  @details This function can be overloaded by the application. It
469  will be called after the RenderScene() loop, before
470  shutting down the render API. It enables the application
471  to release any memory/resources acquired in InitView().
472  @return true for success, false to exit the application
473  *************************************************************************/
474  virtual bool ReleaseView() { return true; };
